# Game Guide

Welcome to the Velodata Dashboard game.

## Objective

Stay on air longer than the other players in your class intake.

## What Counts As On Air

- You can log in.
- Your account can still produce heartbeats.
- You can still use the dashboard permissions your role gives you.

## Class Intake Boundary

- You can only play against users in your own class intake.
- Fake accounts created by students should stay inside the same class intake.
- Staff users may appear in the same User Management table, but they live in a separate system table.

## Roles

- **Admin** can do powerful account-management actions.
- **Creator** can see what is happening and can create accounts.
- **Protector** is intended to defend against dangerous players.
- **Member** has limited power.
- **Spy** is still being developed.

## Online Users

The Online Users view is important. It shows which accounts are currently producing heartbeats.

If a player is banned or deleted, their account should stop producing heartbeats.

## Weekly Difficulty

The game is designed to become harder as the weeks progress.

Each week may introduce new restrictions, protections, or security checks.
